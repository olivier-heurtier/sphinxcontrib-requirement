# sphinxcontrib-requirement
Sphinx extension to manage requirements

import from csv
Export to csv

Configuration

Requirements

    id (ex: SYS-1)
    priority (HIGH, MEDIUM, LOW)
    type (FUNC, NONFUNC)
    subtype (DATA, USECASE, PERFORMANCE, ...)

    source (CONTRACT, WORKSHOP #n, ...)
    subsystem
    target_version
    status (DRAFT, APPROVED, DEPRECATED)
